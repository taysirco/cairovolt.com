Subject: URGENT & CRITICAL: Technical SEO Architecture Overhaul & Index Recovery Plan for CairoVolt.com

To the Antigravity Development & Technical Team,

We are currently executing an aggressive SEO strategy for CairoVolt.com to dominate the Egyptian market. However, we recently experienced a severe "Index Bloat and Purge" anomaly on Google Search Console (indexed pages dropped from 70 to 30). 

After a deep Technical SEO audit, it is clear that Google's algorithms (Helpful Content & SpamBrain) are de-indexing our pages due to critical structural bugs, cross-brand taxonomy contamination, programmatic duplicate content, and URL routing issues.

To recover our indexation and build the Topical Authority required to outrank major competitors like Amazon and Noon, please implement the following Technical Master Plan immediately:

### 🚨 Phase 1: Brand Taxonomy & UI Logic Fixes (CRITICAL)
Google's crawlers are confused by conflicting brand signals on our product pages, destroying our topical relevance.
* The Bug: Major products are cross-contaminating brands. For example, the "Anker 737 Power Bank" currently has a breadcrumb trail showing: `Home / Joyroom / Power Banks / Anker 737 Power Bank`. Additionally, right under the Anker product price, the UI hardcodes `Joyroom ✓ In Stock`.
* The Fix: 
  - Strictly decouple "Anker" and "Joyroom" logic in the database and category trees.
  - Refactor Breadcrumbs to dynamically pull the *exact* and *correct* parent brand/category of the specific product being viewed.
  - Fix the availability text snippet to dynamically fetch the product's actual assigned brand (e.g., it must render `Anker ✓ In Stock` for Anker products).

### 🗑️ Phase 2: Eradicate Programmatic Duplicate Content
Google is flagging our product pages as "Low Quality/Duplicate" because of static template text.
* The Bug: The "Expert Opinion. Cairo Volt Team" section is acting as a static global block. The exact same boilerplate text is copy-pasted across entirely different products (e.g., Smartwatches sharing text with Car Accessories, and Anker pages praising Joyroom's protection technologies).
* The Fix: 
  - Convert the "Expert Opinion" block into a dynamic, product-specific Custom Field (Metafield) in the CMS backend.
  - Wipe/Clear all current duplicated boilerplate text from the database for this field globally.
  - Ensure this field is completely blank by default. If the field is left empty in the backend, the HTML container should NOT render on the frontend. This will allow our content team to manually write 100% unique reviews for every single product.

### 🔗 Phase 3: URL Structure & Server-Level Routing
* The Bug: The system generates URLs with capital letters (e.g., `/en/Joyroom/car-chargers`). This creates case-sensitivity duplicate URL issues on servers and dilutes the crawl budget.
* The Fix: 
  - Force a strict lowercase URL policy across the entire application globally.
  - Implement an automatic 301 redirect rule (via server config) to permanently redirect any uppercase/mixed-case URL requests to their exact lowercase counterparts.

### ⚙️ Phase 4: Core Technical SEO Tags
* Strict Canonicalization: Inject dynamic, self-referencing `<link rel="canonical" href="[Clean_URL]" />` tags in the `<head>` of all pages. Crucially, any URLs generated by sorting/filtering parameters (e.g., `?sort=price`) MUST point their canonical tag back to the clean, parameter-less base URL to prevent index bloat.
* Hreflang Implementation: Since the site utilizes an `/en/` directory, properly implement bi-directional `hreflang` tags in the `<head>` to define the relationship between the Arabic and English localized versions. This prevents Google from treating translations as competing duplicate content.

### 💎 Phase 5: Schema Markup (JSON-LD) Injection
To steal CTR and occupy maximum SERP real estate:
* Product Schema: Deploy dynamic `Product` JSON-LD schema on all single product pages. It must accurately map: `name`, `image`, `description`, `brand`, `offers` (with `price`, `priceCurrency: "EGP"`, `availability: InStock`), and `aggregateRating`.
* FAQ Schema: We have great Q&A sections (e.g., "Does it charge MacBook Pro 16-inch M5?"). Dynamically wrap these QA pairs in valid `FAQPage` JSON-LD schema so they generate rich snippet dropdowns directly in Google Search results.

### 🕸️ Phase 6: XML Sitemap & Crawl Optimization
* Sitemap Deep Clean: Reconfigure the `sitemap.xml` generator. 
  - STRICTLY EXCLUDE: Cart, Checkout, User Accounts, Wishlist, Policy pages, Tags, and parameterized filter URLs.
  - INCLUDE ONLY: Homepage, Main Categories, Clean Single Product URLs, and Blog Posts.
* Internal Linking Enablement: Ensure the Blog CMS allows our editors to easily insert contextual exact-match internal links (Do-Follow hyperlinks) pointing directly to product pages to pass link equity.

### ✅ Post-Deployment Protocol
Once all fixes are deployed to production:
1. Clear all server/CDN caches.
2. Ping the new, clean XML Sitemap to Google Search Console.
3. Notify us immediately so we can execute "Validate Fix" for all unindexed pages in GSC.
