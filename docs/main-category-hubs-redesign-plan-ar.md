# خطة إعادة تصميم الأقسام الرئيسية — CairoVolt

## النتيجة المستهدفة

تحويل صفحات `/anker` و`/joyroom` و`/soundcore` إلى صفحات اختيار سريعة: يفهم الزائر الفرق بين الأقسام خلال ثوانٍ، يصل للقسم المناسب بأقل تردد، وتظل دلالة الصفحة ونصوصها وروابطها قابلة للزحف والفهم دون الاعتماد على JavaScript أو قراءة الصور.

## ترتيب الصفحة

1. عنوان `H1` ووعد واضح خاص بالعلامة.
2. إجابة مختصرة مرئية توضّح ما الذي تبيعه الصفحة ولمن تناسب.
3. إشارات ثقة حقيقية ومحددة.
4. عنوان `H2` للأقسام الفرعية ووصف يساعد على اتخاذ القرار.
5. بطاقات الأقسام المصوّرة بروابط HTML مباشرة.
6. الأكثر مبيعًا كمرحلة تالية بعد اختيار الاحتياج.
7. الأدلة الطويلة والمقارنات والأسئلة المرئية.

هذا الترتيب يخدم نية البحث وقرار الشراء معًا: النص الدلالي يسبق الصور والكتالوج، لكن البطاقات تظل في بداية التجربة البصرية.

## نظام البطاقات

- نص البطاقة يأتي قبل `<picture>` في ترتيب DOM.
- الصورة موضوعة بصريًا عند `inline-end`: يمين الإنجليزية ويسار العربية.
- صور المنتج نفسها لا تُعكس حتى لا تنقلب الشعارات أو المنافذ أو كتابة العبوة.
- البطاقة كاملة رابط واحد واضح، ومساحة اللمس لا تقل عن 44 بكسل.
- الموبايل يستخدم شريطًا أفقيًا مع `scroll-snap`، والتابلت/الديسكتوب يستخدم شبكة ثابتة.
- الارتفاع، نصف قطر الحواف، الظلال، والمسافات موحدة لتقليل الحمل الإدراكي.
- الوصف يجيب عن «متى أختار هذا القسم؟» بدل تكرار اسم القسم.
- شارة قصيرة تعرض عامل قرار حقيقي مثل القدرة أو السعة أو نوع الاستخدام.

## الألوان والحالة النفسية

- الأزرق: للطاقة والثقة والاعتمادية.
- النيلي/البنفسجي: للتقنية والشحن السريع والصوت المتقدم.
- الأخضر النعناعي: للقيادة الهادئة والثبات والأمان.
- الرملي/الخوخي: للكابلات والإكسسوارات اليومية لتقليل الإحساس بالتعقيد.
- الوردي الهادئ: للأجهزة الشخصية والقابلة للارتداء.
- السماوي: للحوامل والتنظيم داخل السيارة.

الخلفيات فاتحة ومنخفضة التشبع حتى تبقى صورة المنتج والنص هما نقطتي التركيز، مع تباين نص لا يعتمد على اللون وحده.

## العربية والإنجليزية

- الصفحة العربية `dir="rtl"` والإنجليزية `dir="ltr"`.
- الاتجاهات مبنية على خصائص منطقية وسلوك RTL، لا على نسخ تصميم منفصل هش.
- سهم الإجراء يتجه مع مسار القراءة في كل لغة.
- أسماء التقنيات والأرقام تبقى مقروءة داخل النص العربي باستخدام صياغة مختلطة محسوبة.
- النص البديل مستقل لكل لغة ويصف المنتج والقسم والسوق دون حشو كلمات مفتاحية.

## الصور

- مخرجان لكل صورة: `480×480` و`800×800` بصيغة WebP.
- الصور شفافة عندما يكون المصدر مقصوصًا بجودة جيدة، وبيضاء عندما يكون المصدر الأصلي صورة منتج بيضاء.
- أسماء الملفات تصف: العلامة + القسم + الاستخدام + CairoVolt + Egypt.
- الصور تُخدم من `/images/categories/...` مباشرة، وليس من مسار API أو رابط مؤقت.
- `srcset` و`sizes` يقدمان المقاس المناسب مع أبعاد صريحة لمنع تغيّر التخطيط.
- إجمالي الـ28 ملفًا الحالي نحو 621 كيلوبايت.

## EXIF/XMP وحقوق المصدر

يضيف خط المعالجة فقط بيانات يمكن إثباتها:

- العلامة المرتبطة بأصل الكتالوج: Anker أو JOYROOM أو Soundcore، من دون اعتبار
  اسم العلامة إثباتًا لمالك حقوق الصورة.
- مساهمة كايرو فولت: إعداد مشتق بطاقة القسم، من دون ادعاء تصوير أصل المنتج.
- حالة الحقوق غير محسومة بهذه البيانات، وشروط الاستخدام لا تمنح إذن إعادة الاستخدام تلقائيًا.
- عنوان ووصف عربي/إنجليزي.
- كلمات موضوعية محدودة مرتبطة بالقسم.
- Digital Source Type: `composite` لأن الملفات مشتقات معدلة من أصول منتجات.

لا تُضاف كاميرا أو عدسة أو تاريخ التقاط أو إحداثيات GPS أو حقول location shown.
وجود Cairo/Egypt في اسم الملف أو وصف السوق لا يدّعي مكان التقاط الصورة أو مكانًا ظاهرًا فيها.

## C2PA

- يوجد تعريف manifest مستقل لكل مخرج في `.c2pa-manifests/category-cards`.
- لا يُضمّن توقيع تطوير ذاتي ويُعرض على أنه موثوق.
- التضمين الإنتاجي يتم فقط عند توفير ملف إعدادات يحتوي اعتماد توقيع إنتاجي:

  `C2PA_SIGNING_SETTINGS=/secure/cairovolt-c2pa.toml npm run assets:categories`

- يعيد خط الإنتاج فحص البيان بعد التوقيع ويرفض النشر عند وجود أي فشل ثقة، بما فيه
  `signingCredential.untrusted`، كما يشترط نتيجة `signingCredential.trusted`.
- يشمل الفحص التوقيع النشط وكل parent ingredient، بما في ذلك `ingredientDeltas`
  و`validation_status` القديمة، فلا يخفي اعتماد جديد موثوق أصلًا غير موثوق.
- يُربط ملف المنتج المصدر كـparent ingredient داخل سلسلة الأصل.
- لكي تتعرف Google على الاعتماد، يجب أن تتسلسل الشهادة إلى جهة إصدار موجودة في
  C2PA Trust List؛ الثقة المحلية داخل جهاز البناء وحدها لا تكفي.
- يُذكر إنشاء المشتق كـ`c2pa.created` ومصدر رقمي `composite`.
- حتى توفير الاعتماد، الصور الحالية غير موقعة، وتبقى ملفات manifest جاهزة للتوقيع فقط.

## SEO وAEO

- `H1` واحد لكل صفحة، ثم `H2` صريح للأقسام.
- أسماء الأقسام ووصفها وروابطها نص HTML مرئي، وليست نصًا داخل الصورة.
- كل بطاقة تستخدم رابطًا canonical داخليًا صالحًا بالعربية والإنجليزية.
- `CollectionPage` يربط صفحة العلامة بالأقسام التابعة عبر `hasPart`.
- `ItemList` يثبت ترتيب الأقسام، و`ImageObject` يصف ملفات الصور المباشرة.
- الصور الجديدة مدرجة في image sitemap لكلتا اللغتين.
- لا تستخدم خريطة الصور حقول موقع أو ترخيص مضللة.
- لا تُخترع تواريخ نشر للمقالات؛ تُرسل فقط عند وجود تاريخ حقيقي.
- صفحة Joyroom Car Accessories تجمع الشواحن والحوامل، بينما يظل رابط كل منتج على تصنيفه canonical الحقيقي.

## CRO والتحويل

- عنوان البطاقة يبدأ بالنتيجة التي يريدها العميل، لا باسم تقني فقط.
- الوصف يقلل خوف عدم التوافق أو سوء الاختيار.
- شارات القدرة/السعة تساعد المقارنة قبل فتح القسم.
- إشارات الأصالة والضمان والدفع عند الاستلام تظهر قبل قرار الاستكشاف.
- زر الأكثر مبيعًا يظل مسارًا سريعًا لمن يعرف أنه يريد الشراء مباشرة.
- لا تستخدم أرقام مبيعات أو ندرة أو ادعاءات ثقة غير قابلة للإثبات داخل البطاقات الجديدة.

## القياس بعد الإطلاق

تُقارن فترة 14–28 يومًا بما قبل الإطلاق مع تقسيم النتائج حسب اللغة والجهاز:

- نسبة النقر من صفحة العلامة إلى قسم فرعي.
- نسبة إضافة للسلة بعد المرور من بطاقة قسم.
- معدل الرجوع من صفحات العلامات.
- زمن الوصول لأول منتج مناسب.
- التحويل لكل قسم ولكل ترتيب بطاقة.
- ظهور صفحات العلامات والأقسام والصور في Search Console.
- Core Web Vitals، خصوصًا LCP وCLS وINP.

لا تُغيّر الألوان والنصوص والترتيب في اختبار واحد. يبدأ الاختبار بترتيب البطاقات أو صياغة العنوان، ثم يُحسم بالتحويل المؤهل وليس النقر وحده.

## قائمة QA قبل النشر

- فحص `/anker` و`/joyroom` و`/soundcore` ونسخ `/en/...`.
- فحص 375 و768 و1024 و1440 بكسل.
- التأكد من عدم انعكاس المنتجات والشعارات في RTL.
- عدم تداخل النص والصورة مع أطول نص عربي.
- عمل لوحة المفاتيح وحلقة التركيز وأسماء الروابط.
- عدم وجود صور مكسورة أو صفحات قسم فارغة.
- فحص JSON-LD وعدم تكرار `CollectionPage`.
- فحص image sitemap والروابط المباشرة.
- تشغيل TypeScript وESLint وNext production build.
- التحقق من EXIF/XMP وC2PA لكل عينة بعد أي إعادة توليد.

## الصيانة

المحتوى التجاري والصور مفصولان عن مصدر التصنيفات. إضافة قسم جديد تتطلب: تعريفه في taxonomy، إضافة عرضه في `category-discovery.ts`، إضافة أصل إلى سكربت الصور، تشغيل `npm run assets:categories`، ثم اجتياز قائمة QA أعلاه.
